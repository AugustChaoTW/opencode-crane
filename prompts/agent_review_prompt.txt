## SECTION 2: Review instructions

### 2a. Feynman Analysis
Call `crane generate_feynman_session` with:
- paper_path: "{{paper_path}}"
- project_dir: "{{project_dir}}"
- mode: "{{feynman_mode}}"  # methodology | post_evaluation | writing
- num_questions: {{num_questions}}
- focus_dimensions: {{focus_dimensions}}

Then apply issue-specific follow-up questions based on the issue content.

### 2b. Le Chun Validation
Check for:
- [ ] Reproducible steps ("## 重現步驟" section)
- [ ] Quantified expected vs actual ("## 預期" / "## 實際" / "## 量化差異")
- [ ] Dependencies declared ("## 依賴")
- [ ] Clear success criteria

---

## SECTION 3: Decision rules (quantified)

### Verdict determination:
- **PASS**: Both Feynman AND Le Chun checks pass
- **NEEDS_WORK**: Any check fails

### Reopen criteria (量化規則):
- Success criteria checklist has unchecked items
- Claimed numbers differ from recomputed values by >5%
- Finding described as "important" but not in any paper
- Logical contradiction exists not tracked by another open issue

## Variables reference:
- {{issue_number}} - GitHub issue number
- {{issue_title}} - Issue title
- {{issue_body}} - Full issue body
- {{issue_comments}} - All comments on the issue
- {{paper_path}} - Path to paper.tex
- {{project_dir}} - Project directory
- {{feynman_mode}} - Feynman mode (default: methodology)
- {{num_questions}} - Number of questions (default: 5)
- {{focus_dimensions}} - Focus areas array

- {{feynman_questions}} - Generated questions list
- {{verdict}} - PASS or NEEDS_WORK
- {{reasoning}} - Detailed reasoning
- {{next_steps}} - Action items if any
